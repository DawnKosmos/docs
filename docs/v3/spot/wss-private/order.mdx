
<p>This topic pushes order information，avg.filledPrice = Z / z</p>
<p><b>Push frequency</b>: real-time</p>
<p><b>Topic</b>: <code>order</code></p>


#### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
| e | string | Event type |
| E | string | Event time |
| s | string | Trading pair |
| c | string | User-generated order ID |
| S | string | BUY indicates buy order, SELL indicates sell order |
| <a href="#order-type-type-ordertypes">o</a> | string | Order type, LIMIT/MARKET_OF_QUOTE/MARKET_OF_BASE |
| <a href="#timeinforce-timeinforce">f</a> | string | <a href="https://www.bybit.com/en-US/help-center/bybitHC_Article?language=en_US&id=000001044">Time in force</a> |
| q | string | Quantity |
| p | string | Price |
| <a href="#order-status-status">X</a> | string | Order status |
| i | string | Order ID |
| M | string | Order ID of the opponent trader |
| l | string | Last filled quantity |
| z | string | Total filled quantity |
| L | string | Last traded price |
| n | string | Trading fee (for a single fill) |
| N | string | Asset type in which fee is paid |
| u | boolean | Is normal trade. False if self-trade. |
| w | boolean | Is working. <code>false</code>：inValid; <code>true</code>：valid |
| m | boolean | Is LIMIT_MAKER |
| O | string | Order creation time |
| Z | string | Total filled value |
| A | string | Account ID of the opponent trader |
| C | boolean | Is close |
| v | string | leverage |
| d | string | NO_LIQ indicates that it is not a liquidation order. IOC indicates that it is a liquidation order. |
| t | string | Trade ID |
