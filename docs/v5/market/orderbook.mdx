Query order book data
* future: A maximum of 200-level of orderbook data can be returned.
* spot: A maximum of 50-level of orderbook data can be returned.
* option: 25-level of orderbook data can be returned ONLY.

:::tip
The response is in the snapshot format.
:::

### HTTP Request
GET `/v5/market/orderbook`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|category |**true** |string |Product type. `spot`,`linear`,`inverse`,`option` |
|symbol |**true** |string |Symbol name|
|limit |false |integer |Limit for data size per page.<li>`spot`: [1,50], default: `50`.</li><li>`linear`&`inverse`: [1,500],default: `25`.</li><li>`option` has 25 ONLY.</li>|

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|s |string |Symbol name |
|b |array |Bid, buyer. Sort by price desc |
|> b[0] |string |Bid price |
|> b[1] |string |Bid size |
|a |array |Ask, seller. Order by price asc |
|> a[0] |string |Ask price |
|> a[1] |string |Ask size |
|ts |integer |The timestamp(ms) that system generates the data |
|u |integer |Update id, is always in sequence [note: 期货、现货、期权里这个u对应到不同档位的wss orderbook] |

### Request Example

<Tabs>
