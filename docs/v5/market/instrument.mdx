Query a list of instruments of online trading pair.
> **Covers: Spot / Linear contract / Inverse contract / Option**

:::caution
* Spot does not support pagination, so `limit`, `cusor` are invalid.
* When query by `baseCoin`, regardless of category=`linear` or `inverse`, the result will have Linear contract and Inverse contract symbols.
:::

### HTTP Request
GET `/v5/market/instruments-info`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |<b>true</b> |string |Product type. `spot`,`linear`,`inverse`,`option`|
|[symbol](../enum#symbol) |false |string |Symbol name|
|baseCoin |false |string |Base coin. _`linear`,`inverse`,`option` ONLY_ |
|limit |false |integer |Limit for data size per page. [1,1000]. default: `500`|
|cursor |false |string |Cursor. Used to pagination|

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|category |string |Product type |
|list |array |Object |
|> symbol |string |Symbol name |
|> [contractType](../enum#contracttype) |string |Contract type. _`linear`,`inverse` ONLY_|
|> [status](../enum#status) |string |Symbol's trading status |
|> baseCoin |string |Base coin e.g., BTC in BTCUSDT |
|> quoteCoin |string |Quote coin e.g., USDT in BTCUSDT |
|> innovation |string |Whether innovation. _`spot` ONLY_. <li>`0`: non innovation</li> <li>`1`: innovative coin, which price is relatively volatile</li> |
|> settleCoin |string |Settle coin e.g., USDC in BTCPERP. `spot` N/A |
|> optionsType |string |Option type. `Call`,`Put`. _`option` ONLY_ |
|> launchTime |string |Symbol launch timestamp (ms). `spot` N/A |
|> deliveryTime |string |Delivery timestamp (ms). `spot` N/A |
|> deliveryFeeRate |string |Delivery fee rate. `spot` N/A |
|> priceScale |string |Price scale. _`linear`,`inverse` ONLY_|
|> fundingInterval |integer |Funding rate interval (minute). _`linear`,`inverse` ONLY_ |
|> leverageFilter |string |Object. _`linear`,`inverse` ONLY_ |
|>> minLeverage |string |Minimum leverage |
|>> maxLeverage |string |Maximum leverage |
|>> leverageStep |string |The minimal step to increase/reduce leverage |
|> priceFilter |string |Object. |
|>> minPrice |string |Minimum order price. `spot` N/A |
|>> maxPrice |string |Maximum order price. `spot` N/A |
|>> tickSize |string |The minimal step to increase/reduce order price |
|> lotSizeFilter |string |Object. |
|>> basePrecision |string |Base coin precision. _`spot` ONLY_ |
|>> quotePrecision |string |Quote coin precision. _`spot` ONLY_ |
|>> minOrderQty |string |Minimum order qty |
|>> maxOrderQty |string |Maximum order qty |
|>> minOrderAmt |string |Minimum order amount. _`spot` ONLY_ |
|>> maxOrderAmt |string |Maximum order amount. _`spot` ONLY_ |
|>> postOnlyMaxOrderQty |string |Maximum order qty for a single `PostOnly` order. _`linear`,`inverse` ONLY_ |
|>> qtyStep |string |The minimal step to increase/reduce order qty |
|> unifiedMarginTrade |boolean |Whether supports to trade under UMA/UTA. _`linear`,`inverse` ONLY_ |
|nextPageCursor |string |Cursor. Used to pagination |
