> **Cover: Option (Unified Account)**

:::tip
This endpoint provides the batch order mode to place a bunch of orders.
- Make sure you have sufficient funds in your account when place an order. Once an order is placed, according to the
funds required by the order, the funds in your account will be frozen by the corresponding amount during the life cycle
of the order.
- The maximum number of orders included in each request is **20** orders. The returned data list is divided into two lists.
The list created by the order and the created information are returned. The structure of the two lists are completely consistent.
:::

### HTTP Request
POST `/v5/order/create-batch`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |**true** |string |Product type. `option` |
|request |**true** |array |Object |
|> symbol |**true** |string |Symbol name|
|> side |**true** |string |`Buy`, `Sell`|
|> orderType |**true** |string |`Market`, `Limit`|
|> qty |**true** |string |Order quantity|
|> price |false |string |Order price. Invalid if `orderType`=_Market_|
|> orderIv |false |string |Implied volatility. `option` ONLY. `orderIv` has a higher priority than `price`. Pass the real value, e.g for 10%, 0.1 should be passed |
|> [timeInForce](../enum#timeinforce) |false |string |<a href="https://www.bybit.com/en-US/help-center/bybitHC_Article?language=en_US&id=000001044">Time in force</a> <li>Market order will use `IOC` directly</li><li>If not passed, `GTC` is used by default</li> |
|> orderLinkId |false |string |User customised order id. A max of 36 characters. Combinations of numbers, letters (upper and lower cases), dashes, and underscores are supported.<br/>_rule of `option`_:<br/><li>**required** param</li><li>always unique</li>|
|> reduceOnly |false |boolean |<a href="https://www.bybit.com/en-US/help-center/bybitHC_Article?language=en_US&id=000001047">What is a reduce-only order?</a> `true` means your position can only reduce in size if this order is triggered. |
|> mmp |false |boolean |Market maker protection. `option` ONLY. `true` means set the order as a market maker protection order |
