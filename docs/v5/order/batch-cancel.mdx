This endpoint provides the batch order mode to cancel a bunch of open orders.
> **Cover: Option (Unified Account)**

:::info important
- You must specify `orderId` or `orderLinkId` when batch cancel.
- If `orderId` and `orderLinkId` is not matched, system will process `orderId` first.
- You can cancel unfilled, partially filled orders.
- A maximum of 20 orders can be amended in per request.
:::

### HTTP Request
POST `/v5/order/cancel-batch`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |**true** |string |Product type. `linear`, `option` |
|request |**true** |array |Object |
|> symbol |**true** |string |Symbol name |
|> orderId |false |string |Order id. Either `orderId` or `orderLinkId` is required |
|> orderLinkId |false |string |User customised order id. Either `orderId` or `orderLinkId` is required |

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|result |Object | |
|> list |array |Object |
|>> category |string |Product type |
|>> symbol |string |Symbol name |
|>> orderId |string |Order id |
|>> orderLinkId |string |User customised order id |
|retExtInfo |Object | |
|> list |array |Object |
|>> code |number |Success/error code |
|>> msg |string |Success/error message |
