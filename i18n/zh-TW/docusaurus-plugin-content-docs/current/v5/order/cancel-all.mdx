撤銷全部委託單
> **統一帳戶覆蓋範圍: 現貨 / 正向合約 / 期權**<br/>
> **普通帳戶覆蓋範圍: 正向永續 / 反向合約**

:::info
- 支持按照`symbol`/`baseCoin`/`settleCoin`撤銷訂單，若您傳入了多個參數組合, 系統僅會處理其中一個參數，其中優先級為`symbol` > `baseCoin` > `settleCoin`.
- **注意**: 當`category`=_option_, 您可以不傳人三個參數中的任何一個，就能取消所有期權的委託單。但是, 對於其他產品, 您必需指定三個參數的其中一個。
:::

### HTTP請求
POST `/v5/order/cancel-all`

### 請求參數
| 參數 | 是否必需 | 類型 | 說明|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |**true** |string |產品類型 <ul><li>統一帳戶: `spot`, `linear`, `option`</li><li>普通帳戶: `linear`, `inverse`. 這裡`category`字段不參與業務邏輯，僅做路由使用，意味著當通過baseCoin來取消全部時，會將正向和反向的訂單都取消掉</li></ul> |
|symbol |false|string |合約名稱 |
|baseCoin |false|string |交易幣種 |
|settleCoin |false|string |結算幣種. 不支持`spot`|
|orderFilter |false |string |僅`spot`有效. `Order`: 普通單,`tpslOrder`: 止盈止損單. 若不傳, 默認是`Order`|

### 響應參數
| 參數 | 類型 | 說明|
|:----- |:-----|----- |
|list |array |Object |
|> orderId |string |訂單Id |
|> orderLinkId |string |用戶自定義訂單Id |

### 請求示例


```http
POST /v5/order/cancel-all HTTP/1.1
Host: api-testnet.bybit.com
X-BAPI-SIGN: XXXXX
X-BAPI-API-KEY: XXXXX
X-BAPI-TIMESTAMP: 1672219779140
X-BAPI-RECV-WINDOW: 5000
Content-Type: application/json

{
  "category": "linear",
  "symbol": null,
  "settleCoin": "USDT"
}
```
