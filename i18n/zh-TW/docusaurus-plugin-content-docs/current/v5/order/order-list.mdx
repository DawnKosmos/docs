Query order history. As order creation/cancellation is **asynchronous**, the data returned from this endpoint may delay. If you want to get
real-time order information, you could query this [endpoint](open-order#) or rely on websocket stream (recommended)
> **Cover: Spot (Unified Account) / Linear Perpetual (Unified Account) / Option (Unified Account)**

:::info
* The orders in 7 days: support to query all status orders
* The orders out of 7 days: support to query those filled orders ONLY
* You can query by symbol, baseCoin, orderId and orderLinkId, and if you pass multiple params, system will process them according to this priority: orderId > orderLinkId > symbol > baseCoin.
:::

### HTTP Request
GET `/v5/order/history`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |**true** |string |Product type. `spot`,`linear`,`option` |
|symbol |false |string |Symbol name|
|baseCoin |false |string |Base coin. `option` ONLY. All option open orders are returned if not passed |
|orderId |false |string |Order id |
|orderLinkId |false |string |User customised order id |
|orderFilter |false |string |`Order`: active order, `StopOrder`: conditional order|
|[orderStatus](../enum#orderstatus) |false |string |Return all status orders if not passed |
|limit |false |number |Limit for data size per page. [1,50]. default: `20`|
|cursor |false |string |Cursor. Used to pagination|
