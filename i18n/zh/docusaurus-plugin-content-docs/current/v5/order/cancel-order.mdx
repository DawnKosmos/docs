> **Cover: Spot (Unified Account) / Linear Perpetual (Unified Account) / Option (Unified Account)**

:::info important
- You must specify `orderId` or `orderLinkId` when cancel the order.
- If `orderId` and `orderLinkId` is not matched, system will process `orderId` first.
- You can cancel unfilled, partially filled orders.
:::

### HTTP Request
POST `/v5/order/cancel`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |**true** |string |Product type. `spot`,`linear`,`option` |
|symbol |**true** |string |Symbol name|
|orderId |false |string |Order id. Either `orderId` or `orderLinkId` is required |
|orderLinkId |false |string |User customised order id. Either `orderId` or `orderLinkId` is required |
|orderFilter |false |string |Valid for `spot` ONLY. `Order`,`tpslOrder`. If not passed, `Order` by default|

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|orderId |string |Order ID |
|orderLinkId |string |User customised order id |
